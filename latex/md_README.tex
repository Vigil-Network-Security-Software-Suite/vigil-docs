Vigil IDS \hypertarget{md_README_autotoc_md13}{}\doxysubsection{Requirements}\label{md_README_autotoc_md13}
Runs well on Debian based linux distributions Requires the following to run\+:
\begin{DoxyItemize}
\item gcc
\item make
\item libpcap-\/dev
\item sqlite3
\item libsqlite3-\/dev
\item liblua-\/5.\+3
\end{DoxyItemize}

Recommened stats for machine running\+:
\begin{DoxyItemize}
\item Recommended 8GB RAM, minimum of 4GB
\item Fast CPU, as many cores as possible
\item Preferably a fast network connection
\item Large hard drive for logs if they are not outsourced
\end{DoxyItemize}\hypertarget{md_README_autotoc_md14}{}\doxysubsection{Features}\label{md_README_autotoc_md14}

\begin{DoxyItemize}
\item Packet capturing using the libpcap library
\item Basic rules for signature based attack identification
\item Logging
\item Remote login shell
\item Statistics 
\end{DoxyItemize}\hypertarget{md_README_autotoc_md15}{}\doxysubsection{Installation}\label{md_README_autotoc_md15}
You just need to run install.\+sh and make Note that if you are not planning on using the web interface and not using a sqlite database, then you need to mark \char`\"{}use\+\_\+sqlite\char`\"{} in the config file as \char`\"{}\+NO\char`\"{}. Please note that the database comes preconfigured and empty with the package.
